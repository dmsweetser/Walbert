Walbert has a lot of trouble understanding that the user_interaction output will conclude its automated behaviors.

And I don't think I like that constraint.

Instead, I think it should be fine to let it reply to the user while also working in the background on the other tasks.

The catch is that the user needs to know it is still working, and that control needs to be returned to the user when no background tasks are included in the LLM response.
