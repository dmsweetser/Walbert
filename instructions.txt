Input and output should be conceptually centralized and unified with one interface. So chat or image input from the user, chat responses, serial input or output, should look similar to Walbert. The idea is that walbert_ blocks determine the next channel to use, and what to provide to it. In theory, Walbert could receive a request from the user, call its smarter cousin, execute a command against a Bluetooth device, get a response, and then return the response in a chat to the user.

Revise the feature-spec.md and implementation-guide.md to reflect these intentions.
