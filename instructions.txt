I had an epiphany.

Get rid of all the code that gives the DB schema to Walbert.

Instead, make everything more rigid, using walbert blocks for the components (for example, persisting an item).

That way, you can base-64 encode and decode the contents more easily.

For the state object for an item, make it JSON that deserializes to a Python object.

Also, change the walbert blocks to use <> so they are valid XML.

Make changes everywhere to accommodate this change in approach. Clean up any unused code.
