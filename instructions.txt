I want you to expand out the walbert_ block format so there are more line types available. The idea is that Walbert should be able to use these to self-determine - for example, like this:

~walbert_should_call_smarter_cousin_start~
YES
~walbert_should_call_smarter_cousin_end~

The idea is that Walbert's response to the user prompt should direct its next steps, which may not include a response to the user yet. And adding nuance to the options available for these walbert_ blocks will give it that flexibility.

Revise the feature-spec.md and implementation-guide.md to account for this. Also, in the implementation-guide.md, include a working system prompt as part of the detail there.
